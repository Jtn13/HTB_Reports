Orion

Machine Easy Linux Web Telnet CVE-2025-32432 2025-06-28

Summary

Orion is a very easy Linux machine that features CSRF Validation Bypass and exploration of CraftCMS and Telnetd. The foothold includes achieving remote code execution by exploiting CVE-2025-32432 in a vulnerable version of CraftCMS. Then the default Craft environment variable file exposes the credentials for its MySQL database, which contains a crackable password. The password has been reused and leads to SSH access to the user on the machine. Finally, privilege escalation is achieved by finding and exploiting a vulnerable version of telnetd (CVE-2026-24061), allowing authentication bypass to root.

Reconnaissance

Nmap

nmap -sV 10.129.244.146

Key ports discovered:

  • 22/tcp - ssh - openSSH
  • 80/tcp - http - nginx

Web Server

Adding orion.htp to etc/hosts we are able to connect to the web server. This displays a site for the "Orion" company and scrolling to the bottom we can find they are using the content managemnet service CraftCMS. Visiting orion.htb/admin we can see that it is running version 5.6.16. Researching this version we can find CVE-2025-32432 which is a remote code execution vulnerability. Using the resources below I was able to learn more about the vulnerability and begin my exploitation.

  • Censys.com
  • Github POC

Finding a User

Now that we have a shell on the web server we are able to look around. Moving back one directory into the ~/html/craft directoy and using ls -la we are able to see a file .env. By running cat .env we are able to see


# Read about configuration, here:
# https://craftcms.com/docs/5.x/configure.html

# The application ID used to to uniquely store session and cache data, mutex locks, and more
CRAFT_APP_ID=CraftCMS--67912ad2-1f1b-4993-bfec-e64daa5c23ff

# The environment Craft is currently running in (dev, staging, production, etc.)
CRAFT_ENVIRONMENT=dev

# General settings
CRAFT_SECURITY_KEY=RRS86F6i2JQKdC6kfEI7frVxA47WVMx8
CRAFT_DEV_MODE=true
CRAFT_ALLOW_ADMIN_CHANGES=true
CRAFT_DISALLOW_ROBOTS=true
CRAFT_DB_DRIVER=mysql
CRAFT_DB_SERVER=127.0.0.1
CRAFT_DB_PORT=3306
CRAFT_DB_DATABASE=orion

CRAFT_DB_PASSWORD=SuperSecureCraft123Pass!

PRIMARY_SITE_URL=http://orion.htb/

Importnantly from this file we can see there is a database with the user root and password SuperSecureCraft123Pass!. Connecting to the database using mysql -u root -p orion and entering in the password we are able to access the database. Running select * from users; we are able to find.


1 | admin | NULL | NULL | NULL | adam@orion.htb | $2y$13$e9zuohgFZzGtbQalcn9Mz.5PJbjxobO0GMbXo8NHp3P/B42LUg0lS | 2026-03-12 11:25:04

This is the database admin and user for CraftCMS but maybe they have reused it for the parent system. However this password is hashed, so we must use hashcat to decrypt it. Since this hash begins with $2y$ it is a bycrpt hash and we can use -m 3200 as the mode.

Running that command hashcat -m 3200 hash.txt /usr/share/wordlists/rockyou.txt we can see that the users password is darkangel.

Now we are able to attempt to login using ssh adam@orion.htb, which successfully connects. In adams home directory we are able to find user.txt which gives us the user flag and the first solve of the box
